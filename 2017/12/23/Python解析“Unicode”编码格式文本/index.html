解决问题

起初在遇到此问题后并没有意识到是log文档本身有问题，而是找了半天代码的问题 = = ，这两句这么简单也不会有什么问题啊。。。于是自己新建一个文本文档尝试读写，哎？竟然成功了，这才意识到可能在文件上面有不同，于是才想到了win下的文本文档有不同的编码格式。。。半天就这么过去了

这里需要注意的是，win新建文本文档时，编码格式默认为ANSI，而我通过工具生成的log文档编码格式是“Unicode”。。。

既然发现了问题，那么接下来就是想办法解决问题了。由于需要开发工具对log进行分析，时间紧迫，于是在度搜索文本编码格式转换工具，很遗憾。没有命令行模式的，不能被我调用。那就只能自己给log文件改编码格式，但是python本身就解析错误，故放弃。Dos嘛。。。文本文档可以被记事本打开，说明解析肯定没有问题，如果我可以在dos下解析，然后重定向到新的txt文档，新的文档为默认编码“ANSI”，对了，用 *type!
